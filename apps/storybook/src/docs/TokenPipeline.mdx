# Token Pipeline

The token pipeline is deterministic and package-oriented:

1. Sanitised fixtures are safety-scanned.
2. Source token JSON is parsed into flat source records.
3. Source paths are mapped into canonical token paths.
4. Canonical tokens are validated.
5. Package outputs are generated from canonical JSON.

The generated output source is `{metadata.source}` and the generated package
contains `{metadata.categories.join(', ')}` categories.

## Generated Artifacts

- `canonical.json`
- `tokens.css`
- `tokens.light.css`
- `tokens.dark.css`
- `index.js`
- `token-names.js`
- `metadata.json`
- `token-docs.json`

Storybook token pages import `token-docs.json`; they do not scrape CSS and do
not read raw fixture files.
