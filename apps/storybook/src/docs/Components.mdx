# Components

The component package is implemented with Mantine internally and uses the
generated theme and token CSS variables supplied by `DemoThemeProvider`.

Apps consume `@demo-ds/components`, not Mantine components or Mantine prop
types. That keeps the public API tied to design-system decisions instead of the
underlying rendering library.

Component-specific values such as Button tone/emphasis colours, StatusBadge
states, AlertBanner tones, Card surface values, and TextInput states come from
generated `component.*` tokens exposed as `--ds-component-*` variables.

## Initial Components

- `Button`: token-backed tone and emphasis mapping.
- `TextInput`: standard description and error behaviour.
- `AlertBanner`: accessible status and alert messaging.
- `Card`: token-backed surface, radius, border, and shadow styling.
- `StatusBadge`: compact semantic state display.
- `PageHeader`: page-level title, description, and actions.
- `ThemeToggle`: design-system colour-scheme integration.
