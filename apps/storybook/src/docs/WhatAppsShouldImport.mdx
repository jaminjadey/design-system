# What Apps Should Import

App code should import:

```ts
import { DemoThemeProvider } from "@demo-ds/mantine-theme";
import "@demo-ds/mantine-theme/styles.css";
import { Button, Card, PageHeader } from "@demo-ds/components";
import { cssVar } from "@demo-ds/tokens";
```

App code should not import:

```ts
import { Button } from "@mantine/core";
```

Mantine is a rendering dependency behind the design-system API. Keeping it
behind the package boundary makes the component contract portable if the
implementation changes later.

Apps generally use semantic token names for app-specific layout. Component
tokens are owned by `@demo-ds/components`; app code should prefer component
props such as `tone`, `emphasis`, and `size` instead of manually applying
`--ds-component-*` variables.

Apps own React and React DOM. The design-system packages own Mantine.

CI enforces this with `pnpm architecture:check` and `pnpm release:check`, which
validate dependency direction, public package subpaths, app imports, public
declaration files, and package manifest policy.
