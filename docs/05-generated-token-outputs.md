# 05 - Generated token outputs

## Output philosophy

All output should come from canonical tokens. Do not generate CSS from raw source files in one script and Mantine output from raw source files in another script. That creates multiple sources of truth.

The generation flow should be:

```txt
source fixtures -> canonical.json -> all generated outputs
```

## Output list

Recommended generated files:

```txt
packages/tokens/dist/
  canonical.json
  index.js
  index.d.ts
  token-names.js
  token-names.d.ts
  tokens.css
  tokens.light.css
  tokens.dark.css
  metadata.json
```

Optional generated files:

```txt
packages/tokens/dist/
  tokens.scss
  token-docs.json
  token-diff.json
```

## CSS variable output

CSS custom properties are the most portable output for the React components and example app.

Recommended naming:

```txt
--ds-color-primitive-primary-10
--ds-color-text-default
--ds-color-background-page
--ds-space-md
--ds-radius-md
--ds-font-size-heading-h1
--ds-line-height-heading-h1
--ds-font-weight-heading-h1
```

Example CSS:

```css
:root,
[data-mantine-color-scheme='light'] {
  --ds-color-text-default: #083344;
  --ds-color-background-page: #f8fafc;
  --ds-space-md: 8px;
  --ds-radius-md: 8px;
}

[data-mantine-color-scheme='dark'] {
  --ds-color-text-default: #fdfdfd;
  --ds-color-background-page: #0f172a;
}
```

Primitive values can live in `:root` if they do not vary by mode:

```css
:root {
  --ds-color-primitive-primary-10: #cffafe;
}
```

## TypeScript output

TypeScript output should support both programmatic use and type-safe token names.

Example generated exports:

```ts
export const tokens = {
  color: {
    semantic: {
      text: {
        default: {
          cssVariable: '--ds-color-text-default',
          value: {
            light: '#083344',
            dark: '#FDFDFD'
          }
        }
      }
    }
  }
} as const;

export type TokenName =
  | 'color.semantic.text.default'
  | 'color.semantic.background.page'
  | 'space.md'
  | 'radius.md';
```

Provide helper functions only if they are genuinely useful:

```ts
export function cssVar(name: TokenName): string {
  return `var(${tokenNameToCssVariable[name]})`;
}
```

## Metadata output

Generate metadata that Storybook and CI can read:

```json
{
  "name": "demo-design-system-tokens",
  "source": "sanitised-design-token-fixtures",
  "tokenCount": 479,
  "modes": ["light", "dark"],
  "categories": ["color", "space", "radius", "typography"]
}
```

Avoid non-deterministic timestamps in committed metadata, or place timestamps in uncommitted reports only.

## Light and dark CSS files

Provide both combined and split CSS output:

```txt
tokens.css        # all tokens, mode selectors included
tokens.light.css  # only light-mode values
tokens.dark.css   # only dark-mode values
```

This helps different consumers:

- Storybook can import `tokens.css`.
- An app that handles mode itself can import the combined file.
- A static preview can import only one mode file.

## Documentation data

Storybook token pages should not scrape CSS. Generate token docs data directly from canonical JSON.

Suggested file:

```txt
packages/tokens/dist/token-docs.json
```

Suggested shape:

```json
{
  "groups": [
    {
      "name": "Semantic text colours",
      "tokens": [
        {
          "name": "color.semantic.text.default",
          "cssVariable": "--ds-color-text-default",
          "value": {
            "light": "#083344",
            "dark": "#FDFDFD"
          }
        }
      ]
    }
  ]
}
```

## Generated file headers

For generated TypeScript and CSS files, include a header:

```txt
Generated by @demo-ds/token-pipeline.
Do not edit directly.
Source: sanitised-design-token-fixtures.
```

For JSON files, put equivalent metadata in a `meta` object.

## Do not expose source paths by default

Canonical tokens may contain source provenance during development, but public package output should avoid exposing source filenames if that becomes noisy or confusing.

Recommended approach:

- Keep source provenance in `canonical.debug.json` for local debugging.
- Keep public `canonical.json` clean.
- Do not include private-looking path names in public docs.

## Snapshot strategy

Snapshot the generated outputs that define the public contract:

- `canonical.json`
- `tokens.css`
- `token-names.ts`
- Mantine theme generated token mapping

Avoid snapshotting files with timestamps, absolute paths, or environment-specific output.

## Output compatibility promise

For the demo, treat these as public contracts:

- Token names.
- CSS variable names.
- Package exports.
- Mantine theme package exports.
- Component package exports.

Changing them should be deliberate and documented.
